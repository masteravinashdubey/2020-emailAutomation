Introduction

The desktop app consists of developing an environment which makes it easier for Elsi team to work with handling the mails and semi-automate the whole Elsi process .
Technology stack includes:

  1. Postgres sql for database
  2. Django for back-end
  3. Electron-vue package for desktop front-end
  4. REST APIâ€™s and Axios for communication
  5. Gmail Api for working with mails

The Git repository link is available here .
This document will give further insights on how to use and reproduce.

PROJECT STRUCTURE:

In vue_frontend/source we have,

views assets
|---- algo.vue |---- states-and-districts.json
|---- aws.vue This json file contains the Indian
|---- cws.vue states with their districts data in json format.
|---- ema.vue
|---- Home.vue
|---- sim.vue

Home.vue --> This file is the homepage of app code of login, logout and setting functionality is provided here .
Sim.vue--> This file contains code of whole implementation of send information mail part.
Aws.vue--> This contains the implementation of announcing the workshop and sending mails to respective colleges, eyantra team members and heads.
Cws.vue--> This file contains the creation of new workshop part.
Algo.vue-->This file contains the whole implementation of selection of team members for workshop based on different selection criteria .
Ema.vue--> This file contains email analytics part.

In django_backend/ folder we have folders :
Assets --> This folder contains some important files like credentials.json which contain the gmail api credentials , all other files have their general respective roles.
Static --> This folder contains all .html files .
Main/views.py--> This file contains the main logic of our backend app.
Main/models.py--> This file defines the database tables structure .
Main/urls.py--> This file contains redirect URI to the respective functions in views.py .

More depth knowledge of the structured frameworks and getting started guide can be found on link1 (electron-vue) and link2 (django) . If you want to get started from beginning these links will help you do so. Detailed information about each function and file will be provided in the file comments itself.
