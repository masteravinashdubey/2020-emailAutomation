Installation Guide

QuickStart: For testing the app

Here is a quickstart guide it is highly recommanded that you first follow this guide and then make changes according to your needs this guide will help you to start the app and then test with dummy data.Few steps will be different for Linux and Windows, so they are accordingly provided.

  1. Clone the git repo. by downloading the zip or by git command.
    By zip:
    • Just download the zip from repo. and extract it in a folder (download link).
    By git commands:
    • To install git on your machine, follow refer this
      Install Git
    • Create a Folder, name it whatever you like.
    • Open the terminal within that folder.
    • Type
      git config --global user.name "John Doe"
      git config --global user.email john@example.com
      Replace name and email with yours
    • Then do,
      • git init
      • git remote add origin https://github.com/SabanwarVivek/5_Email_Automation_and_Spatial_Analysis_A.git
      • git pull origin master
  2. To install python
    For Windows -->
    • Download Python
    • Run command pip install virtualenv in base project directory.
      Setup your virtual environment named 'venv' by writing this command
      virtualenv venv
    • Switch to your virtual environment by:
      change current directory of cmd to cd venv/Scripts
      and then type activate
      Your virtualenv will get activated.
    For Linux -->
    • Download Python
    • Run the command sudo pip install virtualenv in base project directory.
      Setup your virtual environment named 'venv' by writing this command
      virtualenv venv
    • Switch to your virtual environment by:
      type the following command in base project directory
      source venv/bin/activate
      Your virtualenv will get activated.

  3. You need to install the postgres and pgAdmin (https://www.pgadmin.org/ ) for using postgres sql. For full guide on how to install please refer this link.

  4. After installing , open the pgAdmin by searching in search bar for pgadmin, click on it and a new window will open in browser . Click on servers then right click on PostgreSQL select create and then database. Name the database whatever you like.
  5. Now in test_database folder we have a dump file of pgsql.
  6. To import its data into pgAdmin
    For Windows -->
    Open command prompt with administrator rights, also make sure that your environment variables path are set for that particular location if not, then set the environment variables path. Type the following command
    psql –U username dbname < dbexport.pgsql
    where username is the username of pgadmin and dbname is the database name that you want to import data into .
    Example:
    psql –U postgres elsi < C:\example\data.pgsql
    For setting the environment variables if not set already :
    • In Search, search for and then select: System (Control Panel)
    • Click the Advanced system settings link.
    • Click Environment Variables. In the section System Variables, find the PATH environment variable and select it. Click Edit. If the PATH environment variable does not exist, click New.
    • 4. In the Edit System Variable (or New System Variable) window, specify the value of the PATH environment variable. Click OK. Close all remaining windows by clicking OK.( the path value which you need to write here will be C:\Program Files\PostgreSQL\12\lib and C:\Program Files\PostgreSQL\12\bin these locations may change depending on where you have installed the PostreSQL in system).
    • Reopen Command prompt window, and then run the respective command.
    For Linux -->
    Type the command :
    psql -U username -h host dbname < dbexport.pgsql
    Example:
    psql -U posgtres -h localhost elsi < data.pgsql
  7. Now database is ready.
  8. Now run command `pip install -r requirements.txt` in directory django_backend .
  9. In django_backend/app you will see the settings.py file open it.
    There you will find
          DATABASES = {
           'default': {
            'ENGINE': 'django.db.backends.postgresql',
            'NAME': 'elsi',
            'USER':'postgres',
            'PASSWORD':'1234',
            'HOST':'localhost'
           }
         }
    Here ENGINE depicts the database used ,NAME is the database name, USER and PASSWORD are postgres pgAdmin username and password, HOST denotes server.
    Change the NAME according to database name you created and all other fileds respectively.
  10. Now in django_backend folder open cmd and follow with these commands
    • python manage.py makemigrations <appname>
      -->python manage.py makemigrations main
    • python manage.py migrate main
    • for creating superuser for django admin: python manage.py createsuperuser
    • then finally python manage.py runserver <port>
      -->python manage.py runserver 8081
    The backend sever will start at localhost:8081.
    To visit the django admin panel type in browser http://127.0.0.1:8081/admin and enter the superuser credentials.
    Note: If you want to change localhost server on different port just change 8081 with your preferred port and provide the same port in settings tab present in GUI of app .
  11. Now open cmd in vue_frontend.(install nodejs and npm first, for Windows -> link and for Linux -> link)
  12. Type: npm install ,this will install all the dependencies
  13. And then type: npm run electron:serve.
    Note: If you want to run the app in production environment :
    Open the command prompt in vue_frontend folder and follow these steps:
    • npm run electron: build
    We will get an setup file and executable file with some other files in dist_electron and dist_electron/win-unpacked folder.
  14. Test the app with login credentials sun as username and password both or use superuser credentials if you made one.
  15. Test all the functionality of the app.

    Database

    Some important points:

    • If you have a database in sql and want to change it to psql use this git link .
    • If you have a csv and want to import data to pgAdmin database use psql shell with this command
      \copy Public.”table _name” FROM ‘csv_file_location’ DELIMITER ‘,’ CSV HEADER null ‘null’;
      Example:
      \copy Public.”elsi_college_dtls” FROM ‘C:\example\data.csv’ DELIMITER ‘,’ CSV HEADER null ‘null’;

      Sometimes you need to provide encoding details also of the csv file, so just provide extra parameter encoding ‘UTF8’ where encoding may have any other format other than UTF8 depending on the excel version and file type you are using (.csv, .xlsx).

    • If you have .pgsql file and want to import its data into pgAdmin then open command prompt with administrator rights, also make sure that your environment variables path are set for that particular location if not, then set the environment variables path. Type the following command
      psql –U username dbname < dbexport.pgsql
      where username is the username of pgadmin and dbname is the database name that you want to import data into .
      Example:
      psql –U postgres elsi < C:\example\data.pgsql

    Note: Make a database named elsi in pgadmin. A folder named test_database is provided which contains data.pgsql file this is the testing file which can be used for testing the app (for importing this file follow step 3), later on if you want to truncate the tables and import your own data, type one among these commands in psql shell.

    1. If user wants a simple truncate to happen
      TRUNCATE table_name1,table_name2,....;
    2. If user wants not only to truncate but also reset associated sequence generator(auto-incrementing field of the table)
      TRUNCATE TABLE table_name RESTART IDENTITY;
